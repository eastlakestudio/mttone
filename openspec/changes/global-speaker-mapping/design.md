## Context

目前，Mttone 的转写模块在单一会议内部使用 Whisper 生成“Speaker_X”标签。虽然已经在 UI 上提供了将“Speaker_X”映射到当前会议的参会人的功能，但这些声纹特征并未在跨会议级别上与实际人名建立永久联系。同时，Whisper 在多人交谈时存在未断句的情况，需要人工修正与二次切分。

## Goals / Non-Goals

**Goals:**
1. 实现全局说话人管理：不同会议共享同一批联系人作为声纹映射对象。
2. 强化片段编辑能力：支持对单一转写气泡内的文本内容按需手动换行、拆分为两个气泡。
3. 提供统一的全局回顾：在一个独立页面查看全局特定说话人的所有历史发言并回放。

**Non-Goals:**
1. 暂不涉及训练 Whisper 自定义实时声纹识别模型，仅做标记关联。
2. 不涉及非转写结果的音频波形裁剪工具。

## Decisions

1. **跨会议数据模型**: 
   直接利用/扩展现有的 `contacts` 表作为全局声纹人字典。所有 `speech_clips` 的 `contact_id` 作为跨会议的唯一关联外键。
2. **文本分行与切分逻辑**:
   在 UI 上提供菜单或交互触发“从此处拆分”。业务逻辑上将原 `speech_clip` 废弃，新建两条 `speech_clips`，通过字数比例估算前后两段的 `start_time` 和 `end_time`。
3. **音频片段回放交互**:
   在每个转写气泡 UI 上提供播放功能。利用原生播放器指定 `start_time` 至 `end_time` 的范围区间在源音频文件中播放，不需要产生物理音频切片，节约空间。

## Risks / Trade-offs

- **时间戳估算不准 (Risk)**: 按字数比例估算的拆分时间戳无法百分之百匹配实际说话的物理时间。
  - **Mitigation**: 暂作近似处理，后续可引入强制的时间线滑动调整交互。
