# Mttone 整体开发计划 (Dev Plan)

**日期标签：** 2026-06-24_1802
**阶段目标：** 实现 Mttone 本地离线会议系统，重点攻克会议延续性支持与私域知识库 (RAG) 融合。

为了保证开发过程的可控性，我们将整个工程拆分为以下 5 个主要阶段 (Milestones)。每次进入新的子模块开发前，都会基于此路线图出具详细的实施计划 (Plan) 并进行代码变更，完成后输出对应的 Walkthrough。

## 阶段 1：数据持久层重构与会议元数据增强
**目标：** 建立底层数据基座，支持会议的延续以及文档实体的接入。
- **任务 1.1：** 升级 SQLite 数据库结构，修改 `Meeting` 表增加 `ParentMeetingID`、`主题`、`地点` 字段。
- **任务 1.2：** 新增 `Document` 表及 `MeetingDocumentBinding` 关联表。
- **任务 1.3：** 在 Rust 后端 (`db.rs`) 提供上述表的 CRUD Tauri Command 接口。
- **任务 1.4：** 前端 UI 适配：支持在创建/编辑会议时输入详细信息，并选择“延续历史会议”。

## 阶段 2：私域知识库 (RAG) 基建与文档管理
**目标：** 建立私域知识库的上传、解析与本地化存储链路。
- **任务 2.1：** 新建后端模块 `rag.rs`，实现本地文档 (PDF/Word/TXT) 的文本解析提取。
- **任务 2.2：** 引入本地轻量级向量化方案 (如基于 `fastembed-rs` 或调用本地 Ollama Embedding API)，对文本分块进行向量化。
- **任务 2.3：** 将向量特征持久化存储（探索使用 SQLite VSS 扩展或原生方案）。
- **任务 2.4：** 前端开发“私域知识库”管理面板，支持文档的上传、解析进度展示及列表查看。

## 阶段 3：ASR 引擎适配与热词自适应
**目标：** 将私域知识库与端侧 ASR 打通，提升识别准确率。
- **任务 3.1：** 在 `rag.rs` 中实现“关键词/专有名词提取”算法（可利用本地轻量级 NLP 库或 LLM）。
- **任务 3.2：** 修改前端录音交互，在开启录音前，允许用户勾选并绑定此会议的关联文档。
- **任务 3.3：** 在 `audio.rs` 启动 ASR 前，自动从绑定的文档中抽取热词并注入到 ASR 引擎（依赖所选 ASR 框架的 Hotword 支持能力）。

## 阶段 4：大模型 (Ollama) 与 RAG 会后纪要生成
**目标：** 基于会议转写和私域背景知识，生成高质量会议记录。
- **任务 4.1：** 升级 `llm.rs` 模块，新增针对 RAG 检索的逻辑：根据会议文本的上下文，在向量库中召回最匹配的文档段落。
- **任务 4.2：** 设计并封装 RAG Prompt（包含系统预设、召回的知识库背景、原始会议转写文本）。
- **任务 4.3：** 前端 UI 增加“生成总结（深度模式）”按钮，流式接收 Ollama 返回的高质量带有知识增强的会议纪要。

## 阶段 5：声纹人脉库与前端全局联调
**目标：** 跑通端到端功能并优化体验细节。
- **任务 5.1：** 完善声纹特征提取与 Speaker 绑定逻辑，并落库到 `SpeakerProfile`。
- **任务 5.2：** 全局联调：测试长时录音、中断恢复（延续会议）、以及文档挂载的完整链路。
- **任务 5.3：** 修复内存泄露或大模型占用过高的问题（性能功耗优化）。

### 开发工作流约定
在接下来的开发中，我们将严格遵守以下规则：
1. **Plan：** 开始某个具体任务前，在 `doc/dev-process` 下生成例如 `plan_20260624_task1.1.md` 的计划书，并待您确认。
2. **Execute：** 编写代码实现。
3. **Walkthrough：** 实现完毕后，在同目录下输出例如 `walkthrough_20260624_task1.1.md` 的总结文档，记录实现细节和验证结果。
