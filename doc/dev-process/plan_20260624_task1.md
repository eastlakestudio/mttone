# Mttone 开发计划 - 阶段 1：数据持久层重构与会议元数据增强

本文档为开发路线图中**阶段 1** 的具体实施计划。通过此计划重构数据库，以支持后续的“会议延续”以及“知识库挂载”功能。

## User Review Required

- 请确认我们对数据库表结构的变更设计是否满足业务需求（详细字段见下文）。
- 因为我们使用 SQLite 且目前项目仍在早期，我会直接修改 `schema.sql`，如果本地有老数据库，可以直接删除 `.db` 文件重建以应用最新结构，是否接受？

## Proposed Changes

### 1. 数据库结构升级 (`src-tauri/src/db/schema.sql`)
我们将修改现有的 schema 脚本。
#### [MODIFY] [schema.sql](file:///Users/minghualiu/personal/EastlakeStudio/mttone/src-tauri/src/db/schema.sql)
- **更新 `meetings` 表**：
  - 新增 `parent_meeting_id TEXT` 字段，作为外键指向 `meetings(id)`，用于支持上下午的会议延续。
  - 新增 `location TEXT` 字段，用于记录会议地点。
- **新增 `documents` 表**：
  - 用于记录上传的私域知识库文档。包含字段：`id`, `filename`, `file_path`, `text_content`, `created_at`。
- **新增 `meeting_document_bindings` 表**：
  - 多对多关联表，包含 `meeting_id` 和 `document_id`。

### 2. 后端 CRUD 接口 (`src-tauri/src/db/mod.rs`)
#### [MODIFY] [mod.rs](file:///Users/minghualiu/personal/EastlakeStudio/mttone/src-tauri/src/db/mod.rs)
- 更新 `MeetingInfo` 数据结构，加入新字段 `parent_meeting_id` 和 `location`。
- 修改 `create_meeting_cmd` 的参数，支持传入 `parent_meeting_id` 和 `location`。
- 修改 `get_meetings_list_cmd` 等查询，带出新增字段。
- **新增文档相关 Command**：
  - `add_document_cmd`: 记录上传的文档元信息。
  - `get_documents_cmd`: 获取所有文档列表。
  - `bind_document_cmd`: 将指定文档关联到指定会议。
  - `get_meeting_documents_cmd`: 获取某个会议挂载的文档。

### 3. 前端 UI 适配 (`src/App.tsx`)
#### [MODIFY] [App.tsx](file:///Users/minghualiu/personal/EastlakeStudio/mttone/src/App.tsx)
- 在“开始新录音”按钮触发前，增加一个轻量的“会议信息配置弹窗”（包含：会议主题、地点、延续会议选择、关联文档选择）。
- 更新调用 `invoke("create_meeting_cmd")` 的传参。
